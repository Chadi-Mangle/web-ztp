    Utilisation¶

    Premiers pas¶

    Connexion à l'interface¶

    1. Accédez à l'interface web : 

      http://localhost:8000

    2. Connectez-vous avec vos identifiants :

    3. Utilisateur : admin
    4. Mot de passe : (celui défini lors de l'installation)

    Configuration initiale¶

    1. Configurez le serveur DHCP :
    2. Accédez à "Configuration" > "Serveur DHCP"
    3. Définissez les plages d'adresses IP

    4. Configurez les options DHCP

    5. Définissez les templates de configuration :

    6. Accédez à "Configuration" > "Templates"
    7. Créez des templates pour vos équipements

    Gestion des équipements¶

    Ajout d'un équipement¶

    1. Accédez à "Équipements" > "Ajouter"
    2. Remplissez les informations :
    3. Nom de l'équipement
    4. Modèle
    5. Adresse MAC

    6. Template de configuration

    7. Cliquez sur "Enregistrer"

    Configuration automatique (ZTP)¶

    1. Branchez le nouvel équipement au réseau
    2. L'équipement recevra automatiquement :
    3. Une adresse IP via DHCP
    4. Sa configuration via TFTP

    5. Les paramètres réseau nécessaires

    6. Vérifiez le statut dans "Équipements" > "Statut"

    Gestion du réseau¶

    Surveillance¶

    1. Tableau de bord :
    2. Vue d'ensemble des équipements
    3. Statut des configurations

    4. Alertes en temps réel

    5. Logs :

    6. Accédez à "Monitoring" > "Logs"
    7. Filtrez par type d'événement
    8. Exportez les logs si nécessaire

    Maintenance¶

    1. Mise à jour des configurations :
    2. Modifiez les templates
    3. Appliquez aux équipements concernés

    4. Vérifiez les changements

    5. Sauvegarde :

    6. Exportez les configurations
    7. Sauvegardez la base de données
    8. Archivez les logs

    Fonctionnalités avancées¶

    API REST¶

    1. Authentification : 

      curl -X POST http://localhost:8000/api/token/
  -H "Content-Type: application/json"
  -d '{"username":"admin","password":"votre_mot_de_passe"}'

    2. Exemple d'utilisation : 

      # Récupérer la liste des équipements
curl -H "Authorization: Bearer votre_token" http://localhost:8000/api/equipment/

# Ajouter un équipement
curl -X POST http://localhost:8000/api/equipment/
  -H "Authorization: Bearer votre_token"
  -H "Content-Type: application/json"
  -d '{"name":"switch1","model":"Cisco","mac_address":"00:11:22:33:44:55"}'

    Automatisation¶

    1. Scripts Python : 

      from web_ztp.client import WebZTPClient

client = WebZTPClient(api_url="http://localhost:8000/api/")
client.authenticate("admin", "mot_de_passe")

# Ajouter un équipement
equipment = {
    "name": "switch1",
    "model": "Cisco",
    "mac_address": "00:11:22:33:44:55"
}
client.add_equipment(equipment)

    2. Intégration avec d'autres outils :

    3. Ansible
    4. Puppet
    5. Chef

    Problèmes courants¶

    1. Équipement non détecté :
    2. Vérifiez la connexion réseau
    3. Confirmez l'adresse MAC

    4. Vérifiez les logs DHCP

    5. Configuration non appliquée :

    6. Vérifiez le template
    7. Confirmez les permissions TFTP
    8. Consultez les logs système
